Installing Linux: Part 1

Pre-Install Preperations

This blog post will prepare you to install Ubuntu Linux. After you finish this post, move on to the next one (“Installing Linux”) to actually get Linux on your computer.

We will install Ubuntu Linux alongside Windows. This means that you will be able to choose which operating system that you want to use every time that you start your computer. This process of installing 2 operating systems on a computer is known as ‘Dual-booting’.

So how does one install an operating system? You have likely never done this before, since your computer probably came with Windows or MacOS already installed on it. Well, in the case of Ubuntu Linux, we need to follow these steps:

  1. Create some space for Ubuntu on your computer's hard drive

  2. Download and copy a mini-Ubuntu operating system onto a USB drive

  3. Shut down the computer, plug in the USB, and boot (aka start) your computer ‘from’ the USB. This means the computer will use the mini-Ubuntu operating system that's in the USB, instead of using Windows

  4. Follow the on-screen instructions to install Ubuntu onto your computer's hard drive, alongside Windows.

  5. Done! Now, every time you start your computer, you will have the option of starting Ubuntu or Windows.

This is a rough outline of what we will be doing. In this blog post, we will only do the first 2 steps.

Backup

Right now, if you want, you can do a full backup of your computer (to an external hard drive) using the Windows built-in backup software. During the installation process, we will be making changes directly to your hard drive, which risks losing your files, if done incorrectly. But, if you correctly follow all my steps, nothing will go wrong. However, if you want to be extra careful and you have a spare hard drive lying around (or if you have cloud storage), go ahead. I won't be covering the steps to do that, so you will have to follow an online tutorial. Click here for a link to one, but you can look up another tutorial if you want to.

BIOS / UEFI

There are 2 types of firmware (you don't need to know what firmware means) on computers: BIOS and UEFI. Your computer will use 1 of these 2 types. The install process will be slightly different depending on which one your computer uses.You must find out which type your computer uses. Newer computers (made after about 2010) tend to use UEFI. Follow this tutorial to find out. Remeber which type your computer uses.

Disable Fast startup / Secure boot

A small side note: to dual-boot, we will need to disable some features that will interfere with the Linux installation. Namely, fast startup and secure boot.

To disable fast startup go to Control Panel > Hardware and Sound > Power Options > System Settings > Choose what the power buttons do and uncheck the Turn on fast startup box. Click here for a screenshot tutorial.

Secure boot is a feature exclusive to computers using UEFI. If your computer uses UEFI, follow this tutorial to disable secure boot.

Now, grab a USB (at least 4 GB in size), and let's begin!

1. Download Ubuntu

We will begin by starting the download of the mini-Ubuntu operating system that we will be copying onto our USB. Even though this is step 2 (see above), I want you to start the download now because it may take a while (it's around 2.5 GB). Go to this link and click on the download button for Ubuntu 20.04.1 LTS. I know it says you need 25 GB of free hard drive space on that website, but don't worry, the file you are downloading right now is not that big (it's only 2.5 GB). The website is refering to the space needed when installing Ubuntu on your computer's hard drive (in step 4). You DON'T NEED A USB OF 25 GB! It only needs to be 2.5 GB.

2. Make space on your hard drive

In the meantime, while the file is downloading, let's make some space on our hard drive. Now you will need to have about 40 GB of free space minimum. That's because the Ubuntu OS takes up 25 GB and the other 15 GB is for your personal files (although this number can vary from person to person). Now is the time to choose which computer you want to install Linux on, depending on how much free space you have on it.

Before we proceed, you should check whether your hard drive uses MBR or GPT partion tables (again, you don't need to know what that is). These are 2 different types of ‘things’ that a hard drive can use. All you need to know is which one your hard drive uses. To find out follow this tutorial.

Now that you know if the hard drive you want to install Linux on is using MBR or GPT, let's create some space for Ubuntu. Let me explain what we will be doing right now. All hard drives are split into ‘partitions’. You can think of these as sections of your hard drive. Each section can hold files/data. Your hard drive is already split into some partitions. Only 1 operating system can be installed on each partition. We need to create a new partition to install Ubuntu on. To create a new partition, we need to first ‘shrink’ the size of one of the existing partitions. To do this, open up Disk Management (assuming you are using Windows. For MacOS, I think it's called Disk Utility, but you should look up a tutorial online about how to shrink a partition on MacOS. Or you can ask in the Element group chat for help, because I know someone there has already installed Linux on a Mac) by searching in the bottom left corner. It should look something like this:

Now if your hard drive uses MBR, you can have a maximum of 4 primary partitions. A primary partition is just another name for the regular type of partition that you know of. If you use MBR and your hard drive already has 4 or more partitions, then we will have to delete some of them. If your hard drive uses GPT however, there is no limit to how many partitions you can have, so you can skip the rest of this paragraph. If you need to delete a partition, DO NOT delete your windows partition or C drive. Besides those 2, it is likely that you have a manufacturer partiton (like HP Tools, which I had) and a Windows recovery partition. It's best to delete the manufacturer's partition, (and then if you still have 4 or more, delete the Windows recovery partition) BUT WAIT!!!. Before you delete it, you should make a copy of it onto another drive (unless you really think that you will never need it in the future). To make a copy (or ‘clone’ a partition as it's called), follow this tutorial. And this link may be helpful for those of you running old HP laptops like I was. Honestly, you don't really have to clone the partition, but you can if you want. After doing this, you should have 3 partitions remaining.

Right click on the partition you want to shrink (most likely the C drive partition, for most of you), and click on ‘Shrink Volume’ (volume means partition). A window will pop up asking you how much space you want to shrink in MegaBytes. The value you enter here will be the amount of space that you will have to install Linux on. Note that the value is in MB, so pull out your calculator and enter the amount of space that you want. Make sure you have about 40 GB minimum, as this will give you a comfortable amount of room for installing Ubuntu. But if you have more space than that, then about 50-80 GB will be more than enough (but if you have like a 500 GB hard drive, you can do even more!). Notice that there is a maximum limit to how much you can shrink the partition. If you are not able to shrink the partition enough, then follow this tutorial to ‘defragment’ your hard drive, and then try again. If you still don't have enough space, then backup and delete some of your files, then try again. After you have shrank the partition, there will be a new block that says ‘Unallocated’. This is what we want.

3. Burn the ISO to the USB

Alright, now let's check on that Ubuntu file that we were downloading in the background. That file is called an ‘iso’ file (because its file extension is .iso), and it lets us boot the computer from that file. To do this, we need to copy it onto a USB drive. But a regular copy and paste won't work. We gotta write the iso file to the USB as a partition for this to work. The iso file will occupy an entire partition on the USB, instead of being a file inside a partition. This process of copying a file onto a USB drive as a partition is called ‘burning’ the file onto the USB drive. Note that there's no fire involved here, it's just funny terminology used for historical reasons. To burn the iso file onto the USB, we will need third-party software. If you are using Windows, install Rufus from this link, and if you are using MacOS install Etcher from this link. If you are using Rufus, click this link for a video tutorial (stop the video when he starts talking about booting the computer. All you need to do right now is burn the iso), and if you are using Etcher, click this link. BUT WAIT!!! Before you start burning the iso file to your USB, make sure that you have copied all important files off your USB, because your USB will be completely wiped after burning the iso file to it. That means all partitions on the USB will be wiped (aka deleted)!

And … you're done for now! The hardest part of the installation process is behind you! If you have any questions, let me know.

Good job!
